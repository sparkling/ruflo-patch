# Debugging

## Auto-Memory System (ADR-048/049)
- MEMORY.md is auto-generated by curateIndex() from topic files — do NOT write to it directly
- To remember something, write to the appropriate topic file (preferences.md, patterns.md, etc.)
- curateIndex() runs at session-end and regenerates MEMORY.md as a compact 180-line index
- A PostToolUse hook (memory-redirect.mjs) intercepts accidental MEMORY.md writes and reroutes to topic files
- Topic files: preferences.md, patterns.md, debugging.md, architecture.md, performance.md, security.md, swarm-results.md, ruv-projects.md

## Memory Namespace Enforcement (Patches 19-22)
- **Root cause**: MCP defaulted namespace='default', CLI defaulted 'all' — entries in 'patterns' invisible to MCP
- **Threshold**: 0.3 is upstream design (CLI, searchEntries, ADR-024 uses 0.5). Don't change it.
- **Score distribution** (768-dim mpnet): exact match 0.82, related 0.55-0.78, noise floor ~0.47
- **MCP server caches code** — file patches require MCP restart (`claude mcp remove/add` + new session)
- **Patch 19 (Problem A, 6 ops)**: Search → 'all' — issues #1131, #1123
- **Patch 20 (Problem B, 9 ops)**: Write ops require namespace — issue #581
- **Patch 21 (Problem C, 10 ops)**: Read ops — retrieve requires ns, list defaults 'all' — issue #1135
- **Patch 22 (Problem D, 4 ops)**: 'pattern'→'patterns' typo — issue #1136
- **Patch 23 (Problem E, 9 ops)**: Block namespace='all' on store/delete/retrieve — issue #1137
- **MCP 'required' fields are NOT enforced server-side** — always add runtime `if (!namespace) throw`
- **Design basis**: ADR-006 (namespace required on MemoryEntry) + README examples. NOT "ADR-050" (which is the Intelligence Loop label, not a namespace spec).
- **Patch 21a gotcha**: new string must include unique context (e.g. 'Retrieve a value') to avoid false skip from delete section's identical schema
- **Reorganized**: Old patches 19-24 (discovery order) → new 19-22 (problem order A-D). Old 22→19c-e, old 23→split 20h-i/21h, old 24→21i-j, old 20g→22a-d.
- **Patch 23 merged into 20/21**: The `|| namespace === 'all'` check is built into 20b/d/e/g/h/i and 21b/f/h `new` strings, so Patch 23 ops always skip (safety net). This avoids WARN on idempotent re-runs.
